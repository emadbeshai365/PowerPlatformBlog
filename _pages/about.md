I'm Emad Beshai, a Software Engineer focused on distributed systems, backend architecture, and building software that scales.

This blog is where I share hard-won lessons from production systems &mdash; the failure modes nobody documents, the trade-offs that actually matter, and the engineering decisions I've had to live with long-term.

## What I Write About

- **System Design** &mdash; architecture trade-offs, scalability patterns, and the decisions you have to make before you hit the wall.
- **Backend Engineering** &mdash; service design, API contracts, event-driven systems.
- **Databases** &mdash; PostgreSQL internals, query optimisation, schema migration patterns.
- **Cloud & DevOps** &mdash; deployment strategies, observability, operational reliability.
- **Algorithms & Security** &mdash; practical implementations with real-world context.

## Background

I've worked on systems processing millions of requests per day, spanning microservices, monoliths, and everything in between. I've been on the wrong end of enough production incidents to have strong opinions about which complexity is worth paying for.
