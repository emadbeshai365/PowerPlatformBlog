# Technical Blog Plan &mdash; emadbeshai365.github.io

## Platform
- **GitHub Pages + Jekyll** (native build, zero CI/CD setup)
- **URL:** https://emadbeshai365.github.io

---

## Architecture

| Layer | Choice | Why |
|-------|--------|-----|
| SSG | Jekyll (built-in to GH Pages) | Zero config deployment, Liquid templates, Markdown posts |
| Admin | Decap CMS at `/admin` | Browser-based, commits Markdown to Git, no server needed |
| Comments | Giscus (GitHub Discussions) | Free, no tracking, matches developer audience |
| Search | Client-side lunr.js | No server, fast, works on static sites |
| Analytics | Plausible / Umami | Privacy-friendly, won't hurt SEO |
| Plugins | jekyll-seo-tag, jekyll-feed, jekyll-sitemap | SEO, RSS, sitemap auto-generation |

## Base Design

Based on the approved Sonnet template with these foundations:

- **Fonts:** Cormorant Garamond (display) + Source Serif 4 (body) + Fira Code (mono)
- **Dark/light toggle** persisted to localStorage
- **CSS custom properties** for all design tokens
- **Responsive:** posts column + sidebar → single column on mobile
- **Accessibility:** skip nav, ARIA roles, focus ring, reading progress bar

## SEO Strategy

| Factor | Implementation |
|--------|---------------|
| Title tags | Unique per-page via front matter &mdash; `jekyll-seo-tag` |
| Meta description | `description:` field in post front matter |
| Canonical URLs | Auto-generated by `jekyll-seo-tag` |
| Open Graph + Twitter Cards | Auto-generated from front matter |
| JSON-LD | `BlogPosting` per post, `Blog` on index, breadcrumbs |
| Sitemap | `jekyll-sitemap` plugin &mdash; submit to Google Search Console |
| RSS Feed | `jekyll-feed` plugin |
| Semantic HTML | `<article>`, `<nav>`, `<main>`, `<time datetime>` |
| Performance | `font-display:swap`, no heavy frameworks, inline critical CSS |
| URL structure | Clean: `/multi-tenant-architecture/` via `permalink: /:title/` |
| Dates | `datePublished` + `dateModified` in front matter and JSON-LD |
| Core Web Vitals | No layout shift, fast LCP, explicit image dimensions |

## Admin Panel &mdash; Decap CMS

- **Access:** `emadbeshai365.github.io/admin`
- **Auth:** GitHub OAuth
- **Workflow:** Editor fills form → saves → Git commit to `_posts/` → GH Pages auto-deploys

### Content Model (`admin/config.yml`)

```yaml
backend:
  name: github
  repo: emadbeshai365/emadbeshai365.github.io
  branch: main

media_folder: "assets/images/uploads"
public_folder: "/assets/images/uploads"

collections:
  - name: "posts"
    label: "Blog Posts"
    folder: "_posts"
    create: true
    slug: "{{year}}-{{month}}-{{day}}-{{slug}}"
    fields:
      - { label: "Layout",      name: "layout",      widget: "hidden",   default: "post" }
      - { label: "Title",       name: "title",       widget: "string" }
      - { label: "Date",        name: "date",        widget: "datetime" }
      - { label: "Description", name: "description", widget: "text" }
      - { label: "Topic",       name: "category",    widget: "select",
          options: ["System Design","Backend","Databases","Cloud & DevOps","Algorithms","API Design"] }
      - { label: "Tags",        name: "tags",        widget: "list" }
      - { label: "Read Time",   name: "read_time",   widget: "string" }
      - { label: "Featured",    name: "featured",    widget: "boolean",  default: false }
      - { label: "Series",      name: "series",      widget: "string",   required: false }
      - { label: "OG Image",    name: "image",       widget: "image",    required: false }
      - { label: "Body",        name: "body",        widget: "markdown" }
```
